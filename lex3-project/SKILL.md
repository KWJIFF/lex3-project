## 五、各Agent功能定义

### lex3-project（阶段1-2核心）

**内容生产线** (13节点)：
- 热点追踪 → 选题评估 → 素材搜集 → 结构构建
- 公众号写作 → 小红书写作 → 知乎写作
- 配图设计（含ai-image调用）→ 初稿审核 → 最终审核 → 多平台发布 → 数据复盘

**案件交付线** (8节点)：
- 案件接收 → 案件初筛 → 律师匹配 → 阅卷分析
- 辩护方案 → 沟通与会见 → 质量控制 → 归档复盘

### lex3-research（阶段1-8全流程）

**工作流** (8节点)：
- 政策追踪 → 法规解读 → 案例收集 → 数据清洗
- 深度分析 → 数据库建设 → 报告输出 → 迭代更新

**输出**：
- 素材数据（给lex3-project）
- 研究报告（专业展示）
- 案例汇编（文章引用）

### lex3-video（阶段1-2内容补充）

**工作流** (8节点)：
- 需求分析 → 选题策划 → 脚本创作 → 素材准备
- 视频制作 → 审核修改 → 平台发布 → 数据复盘

### lex3-data（阶段3-4）

**工作流** (6节点)：
- 数据收集 → 数据清洗 → 数据结构化
- 数据分析 → 数据应用 → 反馈优化

### lex3-tech（阶段5）

**工作流** (6节点)：
- 需求分析 → 产品设计 → 开发实施
- 测试优化 → 产品交付 → 迭代升级

### lex3-service（阶段6）

**工作流** (5节点)：
- 市场拓展-律师 → 市场拓展-公检法
- 服务交付 → 客户管理 → 生态建设

### lex3-expansion（阶段7-8）

**工作流** (5节点)：
- 市场研究 → 战略规划 → 业务试点
- 规模扩展 → 生态整合

### lex3-coordinator（全程协调）

**工作流** (6节点)：
- 需求识别 → 阶段判断 → Agent分发
- 执行监控 → 数据协调 → 结果反馈

## 六、Agent调用关系

### 内容生产调用

```
lex3-project需要素材时:
  → lex3-research请求素材
  → lex3-research返回案例/政策/数据
  → lex3-project使用素材创作
```

### 案件分析调用

```
lex3-project案件需要深度分析时:
  → lex3-research请求案例分析
  → lex3-research返回分析报告
  → lex3-project使用分析结果
```

### 研究输出

```
lex3-research产出报告时:
  → 一份给lex3-project做内容素材
  → 一份直接发布打造专业性
  → 一份存入lex3-data做数据沉淀
```

### 协调分发

```
用户需求进入:
  → lex3-coordinator识别需求类型
  → lex3-coordinator判断当前阶段
  → lex3-coordinator分发到对应Agent
  → lex3-coordinator监控执行
  → lex3-coordinator反馈结果
```

## 八、事实核查与质量标准（核心机制）

### 8.1 事实核查机制（对抗AI幻觉）

**每个动作必须包含的事实核查步骤**：

```
标准动作模板：
[X.X.X 执行动作]
[X.X.X 事实核查1：数据来源验证]
[X.X.X 事实核查2：多源交叉验证]
[X.X.X 事实核查3：时效性验证]
[X.X.X 事实核查4：逻辑一致性检查]
```

**事实核查清单**：

| 核查项 | 标准 | 不通过处理 |
|--------|------|------------|
| 数据来源 | 官方/权威来源 | 标记并排除 |
| 交叉验证 | 至少2个独立来源 | 标记为"待确认" |
| 时效性 | 1年内数据/案例 | 标注时间 |
| 逻辑一致性 | 前后逻辑通顺 | 重新推导 |
| 引用准确性 | 直接引用原文 | 核对原文 |
| 法律准确性 | 法条引用准确 | 请律师复核 |

**AI幻觉识别信号**：

| 信号 | 应对 |
|------|------|
| 具体数据过于精确 | 核实数据来源 |
| 法条描述过于简略 | 核对法条原文 |
| 案例细节过于丰富 | 核实案例真实性 |
| 绝对化表述 | 添加限定词 |
| 缺乏具体来源 | 要求标注来源 |

### 8.2 内容质量标准

**专业性标准**：

| 维度 | 标准 | 检验方法 |
|------|------|----------|
| 法律准确 | 法条引用准确 | 请律师复核 |
| 数据准确 | 数据来源可靠 | 交叉验证 |
| 逻辑严密 | 论证链条完整 | 自检逻辑 |
| 观点客观 | 不偏不倚 | 多角度呈现 |
| 实用价值 | 读者有收获 | 价值检验 |

**审美标准**：

| 维度 | 标准 | 检验方法 |
|------|------|----------|
| 结构清晰 | 层次分明 | 结构自检 |
| 表达流畅 | 阅读体验好 | 朗读自检 |
| 视觉美观 | 配图协调 | 设计自检 |
| 风格统一 | 品牌一致 | 对比历史 |
| 细节精致 | 无错别字/格式错误 | 校对自检 |

### 8.3 产出品质分级

**S级（优秀）**：
- 事实100%准确
- 专业深度足够
- 审美精致
- 可直接发布

**A级（良好）**：
- 事实95%准确
- 专业深度尚可
- 审美良好
- 轻微修改后可发布

**B级（可用）**：
- 事实90%准确
- 专业深度一般
- 审美一般
- 需要修改后发布

**C级（不可用）**：
- 事实准确率<90%
- 专业深度不足
- 审美差
- 需要重做
