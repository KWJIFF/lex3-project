# Lex3 服务拓展

## 功能

技术产品市场化。

## 工作流

市场拓展-律师 → 市场拓展-公检法 → 服务交付 → 客户管理 → 生态建设

---

## 节点详解

### Node 1: 市场拓展-律师（20动作）

```
1.1.1 制定拓展计划：
  - 目标律所
  - 目标律师
  - 时间安排
1.1.2 识别目标：
  - 刑事专业律所
  - 金融犯罪专业律师
  - Web3相关律师
1.1.3 收集信息：
  - 律所信息
  - 律师信息
  - 业务特点
1.1.4 初步接触：
  - 电话/邮件
  - 介绍产品
  - 了解需求
1.1.5 记录沟通：
  - 沟通内容
  - 反馈
  - 下一步
1.1.6 筛选高意向：
  - 评估意向度
  - 重点跟进
1.1.7 准备演示：
  - 演示PPT
  - 产品介绍
  - 案例展示
1.1.8 安排演示：
  - 时间预约
  - 线上/线下
  - 参与人员
1.1.9 演示后跟进：
  - 了解反馈
  - 答疑
  - 推进
1.1.10 商务洽谈：
  - 报价
  - 合同条款
  - 合作方式
1.1.11 合同签订：
  - 合同审核
  - 签署
  - 归档
1.1.12 客户入驻：
  - 账号开通
  - 培训
  - 开始使用
1.1.13 定期回访：
  - 使用情况
  - 问题
  - 需求
1.1.14 问题处理：
  - 及时响应
  - 解决
1.1.15 续费跟进：
  - 合同到期前跟进
  - 续费优惠
1.1.16 收集案例：
  - 成功案例
  - 使用效果
1.1.17 口碑推荐：
  - 鼓励转介绍
  - 奖励机制
1.1.18 参加活动：
  - 行业会议
  - 律所活动
1.1.19 拓展渠道：
  - 律所合作
  - 律师社群
1.1.20 分析数据：
  - 转化率
  - 成本
  - 效果
```
