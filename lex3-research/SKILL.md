# Lex3 研究系统

## 功能定位

- **内容生产的素材库**：为lex3-project提供最新案例、政策、数据支撑
- **专业性的展示窗口**：出具专业研究报告，打造行业影响力
- **持续迭代**：不断更新完善素材库，保障准确性和新颖性

---

## 核心原则

### 1. 事实核查（对抗AI幻觉）

每个数据/案例/政策必须经过4层核查：

```
数据收集 → 来源验证 → 交叉验证 → 时效验证 → 逻辑验证 → 入库
```

| 核查层 | 内容 | 不通过处理 |
|--------|------|------------|
| 来源验证 | 官方/权威来源 | 排除 |
| 交叉验证 | 至少2个独立来源 | 标记待确认 |
| 时效验证 | 1年内数据/案例 | 标注时间 |
| 逻辑验证 | 前后逻辑一致 | 重新核实 |

### 2. 准确性标准

| 标准 | 要求 |
|------|------|
| 政策 | 必须来自官方网站 |
| 案例 | 必须核实案号真实性 |
| 数据 | 必须标注来源和时间 |
| 法条 | 必须核对法条原文 |

### 3. 新颖性标准

| 标准 | 要求 |
|------|------|
| 更新频率 | 政策24h内更新 |
| 优先处理 | 热点话题优先 |
| 快速推送 | 新数据主动推送 |
| 时效标记 | 标注数据时效 |

### 4. 专业性标准

| 标准 | 要求 |
|------|------|
| 深度分析 | 多角度论证 |
| 数据支撑 | 用数据说话 |
| 逻辑严密 | 论证链条完整 |
| 实用价值 | 可指导实践 |

### 5. 审美标准

| 标准 | 要求 |
|------|------|
| 报告结构 | 层次清晰 |
| 可视化 | 图表美观 |
| 表达 | 流畅专业 |
| 细节 | 精致无错 |

---

## 架构

```
lex3-coordinator（协调中心）
     ↓
lex3-project（内容生产）
     ↓ 调用
lex3-research（研究）
     ↓ 输出
① 素材数据（给lex3-project内容生产用）
② 研究报告（给专业展示用）
     ↓
lex3-data（数据沉淀）
     ↑ 迭代
   持续更新
```

---

## 与其他Agent的关系

```
调用关系:
lex3-project需要素材 → lex3-coordinator分发 → lex3-research提供素材 → lex3-project使用

协作关系:
lex3-project(内容) ↔ lex3-research(研究) ↔ lex3-data(数据)

输出关系:
lex3-research → 素材库 → lex3-project → 内容发布
lex3-research → 报告 → 公众号/知乎发布 → 专业性打造
```

---

## 工作流

```
政策追踪 → 法规解读 → 案例收集 → 数据清洗 → 深度分析 → 数据库建设 → 报告输出 → 迭代更新
```
