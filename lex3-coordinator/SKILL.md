# Lex3 协调中心

## 功能

全阶段协调：需求识别 → 阶段判断 → 任务分发 → 执行监控 → 数据协调 → 结果反馈

---

## 工作流详解

### Node 1: 需求识别（5动作）

```
1.1.1 接收用户输入（文字/语音）
1.1.2 提取关键词，判断需求类型
1.1.3 识别用户意图（明确/模糊）
1.1.4 判断紧急程度（高/中/低）
1.1.5 输出需求识别结果
```

**需求类型映射**:
| 需求 | 关键词 | 调用Agent |
|------|--------|-----------|
| 内容生产 | "写文章"/"发公众号"/"做小红书" | lex3-project |
| 视频制作 | "做视频"/"短视频" | lex3-video |
| 案件咨询 | "案件"/"咨询"/"委托" | lex3-project(案件线) |
| 研究分析 | "案例"/"政策"/"报告"/"研究" | lex3-research |
| 数据需求 | "数据"/"统计"/"报告" | lex3-data |
| 技术需求 | "开发"/"工具"/"系统" | lex3-tech |
| 服务需求 | "律师服务"/"外包" | lex3-service |
| 扩展方向 | "RWA"/"新领域"/"生态" | lex3-expansion |

### Node 2: 阶段判断（5动作）

```
2.2.1 检查各维度指标：
  - 案源数量（咨询数/月）
  - 律师数量（合作律师数）
  - 数据量（案件数据/内容数据）
2.2.2 对比阶段阈值
2.2.3 确定当前阶段
2.2.4 确定需要启用的Agent
2.2.5 输出阶段判断结果
```

**阶段阈值**:
| 阶段 | 案源 | 律师 | 数据 | 启用Agent |
|------|------|------|------|-----------|
| 1-2 | <5 | <5 | <20 | lex3-project + lex3-video + lex3-research |
| 3 | 5-15 | 5-15 | 20-50 | +lex3-data |
| 4 | 15-30 | 15-30 | 50-100 | +lex3-tech |
| 5 | 30+ | 30+ | 100+ | +lex3-service |
| 6 | 产品完成 | - | - | +lex3-expansion |

### Node 3: Agent分发（8动作）

```
3.3.1 根据阶段判断结果，选择主Agent
3.3.2 根据任务性质，判断是否需要多Agent协作
3.3.3 如需多Agent协作，确定协作顺序
3.3.4 准备传递给Agent的输入数据
3.3.5 调用对应Agent执行任务
3.3.6 记录任务分配信息
3.3.7 设置预期完成时间
3.3.8 发送任务给Agent
```

### Node 4: 执行监控（10动作）

```
4.4.1 定期检查各Agent执行状态
4.4.2 监控任务进度：
  - 是否按时开始
  - 当前进度
  - 是否按时完成
4.4.3 识别执行异常：
  - 任务卡顿
  - 输出不符合预期
  - 技术故障
4.4.4 如发现异常，触发处理机制
4.4.5 记录执行日志
4.4.6 汇总各Agent产出
4.4.7 检查产出是否符合预期
4.4.8 如不符合，反馈给Agent调整
4.4.9 确认任务完成
4.4.10 更新任务状态
```

### Node 5: 数据协调（6动作）

```
5.5.1 监听各Agent的产出事件
5.5.2 判断产出是否需要传递给其他Agent
5.5.3 如需要，提取相关数据
5.5.4 传递给目标Agent
5.5.5 确认数据接收成功
5.5.6 记录数据流转日志
```

**数据流转规则**:
```
lex3-project(内容) → 用户咨询 → lex3-project(案件)
lex3-project(案件) → 案例数据 → lex3-research(研究)
lex3-research(素材) → 素材库 → lex3-project(内容生产)
lex3-research(报告) → 发布 → 专业性
lex3-data(分析) → 技术需求 → lex3-tech(开发)
lex3-tech(产品) → 完成 → lex3-service(市场化)
lex3-service(市场) → 新方向 → lex3-expansion(扩展)
```

### Node 6: 结果反馈（5动作）

```
6.6.1 整理各Agent的执行结果
6.6.2 按用户需求组织输出格式
6.6.3 将结果反馈给用户
6.6.4 如用户有反馈，记录并处理
6.6.5 更新项目状态
```

## Agent调用逻辑

### 内容生产流程

```
用户: "写一篇NFT诈骗的文章"
  ↓
lex3-coordinator 识别: 内容生产需求
  ↓
lex3-coordinator 检查: 当前阶段1-2
  ↓
lex3-coordinator 分发: lex3-project + lex3-research
  ↓
并行执行:
  - lex3-project: 执行内容生产线
  - lex3-research: 提供NFT诈骗案例素材
  ↓
lex3-project 使用素材完成内容
  ↓
lex3-coordinator 汇总结果
  ↓
输出给用户
```

### 研究输出流程

```
用户: "出一份虚拟货币案例报告"
  ↓
lex3-coordinator 识别: 研究需求
  ↓
lex3-coordinator 分发: lex3-research
  ↓
lex3-research 执行:
  - 政策追踪
  - 案例收集
  - 深度分析
  - 报告输出
  ↓
输出:
  - 报告给用户
  - 素材存库
  ↓
lex3-coordinator 确认完成
```
