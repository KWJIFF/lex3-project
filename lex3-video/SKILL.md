# Lex3 视频制作

## 功能

从内容策划到视频发布的完整视频工作流。

## 核心原则

### 1. 内容真实性（对抗AI幻觉）

| 核查项 | 方法 |
|--------|------|
| 法律知识 | 请律师复核 |
| 数据引用 | 核实数据来源 |
| 案例描述 | 脱敏处理+核实 |
| 法规引用 | 核对原文 |

### 2. 专业性标准

| 维度 | 要求 |
|------|------|
| 法律准确 | 律师复核 |
| 逻辑清晰 | 结构自检 |
| 价值明确 | 观众有收获 |

### 3. 审美标准

| 维度 | 要求 |
|------|------|
| 视觉 | 封面+构图专业 |
| 听觉 | 配乐+音质 |
| 节奏 | 节奏紧凑 |
| 细节 | 精致无错 |

### 4. 品质分级

| 级别 | 标准 |
|------|------|
| S级 | 直接发布 |
| A级 | 轻微修改 |
| B级 | 需要修改 |
| C级 | 需要重做 |

---

## 工作流

需求分析 → 选题策划 → 脚本创作 → 素材准备 → 视频制作 → 审核修改 → 平台发布 → 数据复盘

---

## 节点详解

### Node 1: 需求分析（10动作）

```
1.1.1 接收用户需求：视频主题、目标、平台、时长
1.1.2 了解内容来源：文章/案例/热点
1.1.3 确定视频目标：品牌曝光/案源转化/知识普及
1.1.4 评估复杂度：单人口播/访谈/动画/实拍
1.1.5 评估资源需求：设备/人员/场地
1.1.6 评估时间需求：制作周期
1.1.7 评估预算：成本范围
1.1.8 给出建议：形式/时长/风格
1.1.9 确认方案
1.1.10 启动流程
```

### Node 2: 选题策划（12动作）

```
2.2.1 从lex3-project素材库中选择候选选题
2.2.2 评估选题热度：搜索指数/话题讨论
2.2.3 评估选题适配性：是否适合视频形式
2.2.4 评估选题转化性：能否引导咨询
2.2.5 确定视频角度：法律解读/案例分析/风险警示
2.2.6 设计视频结构：开头/主体/结尾
2.2.7 设计视频标题：吸引且准确
2.2.8 设计视频封面：风格统一
2.2.9 确定关键词：便于搜索推荐
2.2.10 撰写视频策划案
2.2.11 提交lex3-project审核
2.2.12 根据反馈调整
```

### Node 3: 脚本创作（15动作）

```
3.3.1 开始撰写脚本
3.3.2 设计开场白（10-20秒）：
  - 痛点切入/悬念设置/数据吸引
3.3.3 写正文第一部分（1-2分钟）：
  - 背景介绍/问题定义
3.3.4 写正文第二部分（2-3分钟）：
  - 核心内容/案例分析
3.3.5 写正文第三部分（1-2分钟）：
  - 法律解读/风险提示
3.3.6 写结尾（10-30秒）：
  - 总结/引导互动/咨询引导
3.3.7 设计镜头指示：
  - 近景/远景/切换
3.3.8 设计配音指示：
  - 语速/语调/情感
3.3.9 设计音效指示：
  - 背景音乐/音效
3.3.10 设计字幕指示：
  - 样式/位置/动画
3.3.11 检查逻辑：论证是否严密
3.3.12 检查节奏：时长是否合适
3.3.13 检查时长：控制在目标范围内
3.3.14 优化脚本
3.3.15 提交审核
```

### Node 4: 素材准备（18动作）

```
4.4.1 根据脚本列出素材清单
4.4.2 收集自有素材：
  - lex3-project历史文章图片
  - 案件素材脱敏图片
4.4.3 搜索免费素材：
  - Pexels/Pixabay
  - Unsplash
4.4.4 搜索付费素材：
  - 视觉中国/站酷
4.4.5 制作数据图表：
  - 使用Canva/Excel
  - 标注数据来源
4.4.6 制作演示动画：
  - 使用PPT/Canva
  - 设计动画效果
4.4.7 调用ai-image生成配图：⭐整合
  - 生成视频封面图
  - 生成视频内插图
  - 输入Prompt：主题、风格、色彩要求
  - 检查质量，不满意则重新生成
4.4.8 录制配音：
  - 使用TTS或录音
  - 检查音质
4.4.9 选择背景音乐：
  - 符合视频风格
  - 无版权问题
4.4.10 整理素材：
  - 按镜头顺序排列
  - 命名规范
4.4.11 检查版权：
  - 图片/音乐/字体
4.4.12 检查质量：
  - 分辨率/格式
4.4.13 素材预处理：
  - 调色/裁剪
4.4.14 分类存储
4.4.15 准备备份
4.4.16 检查完整性
4.4.17 对照脚本检查
4.4.18 标记关键素材
4.4.19 确认完成
```

### Node 5: 视频制作（20动作）

```
5.5.1 创建项目：
  - 打开剪映/PR
  - 新建项目
5.5.2 粗剪：
  - 按脚本顺序排列素材
  - 剪切多余部分
5.5.3 精剪：
  - 调整镜头时长
  - 添加过渡
5.5.4 添加转场：
  - 选择合适转场效果
  - 调整时长
5.5.5 添加配音：
  - 导入配音文件
  - 对齐口型
5.5.6 添加音乐：
  - 调整音量
  - 淡入淡出
5.5.7 添加音效：
  - 关键节点音效
  - 调整音量
5.5.8 添加字幕：
  - 识别或手动添加
  - 校对错字
5.5.9 添加特效：
  - 动态贴纸
  - 花字
5.5.10 添加动画：
  - 图表动画
  - 元素动画
5.5.11 调色：
  - 统一色调
  - 调整亮度对比度
5.5.12 配图：
  - 添加封面图
  - 添加文中配图
5.5.13 抠图（如需要）：
  - 使用AI抠图
  - 调整边缘
5.5.14 合成：
  - 合并所有元素
  - 检查整体效果
5.5.15 自检：
  - 检查逻辑
  - 检查时长
  - 检查质量
5.5.16 渲染输出：
  - 选择格式和画质
  - 等待渲染完成
5.5.17 检查质量：
  - 观看完整视频
  - 检查问题
5.5.18 生成多版本：
  - 抖音版/视频号版/B站版
  - 调整时长和尺寸
5.5.19 整理项目：
  - 保存工程文件
  - 归档素材
5.5.20 归档素材
```

### Node 6: 审核修改（10动作）

```
6.6.1 自检清单：
  [ ] 内容准确：法律知识正确
  [ ] 逻辑通顺：论证清晰
  [ ] 时长合适：不超时
  [ ] 画面清晰：无模糊
  [ ] 声音清晰：无噪音
  [ ] 字幕正确：无错字
  [ ] 版权合规：无侵权
6.6.2 如有问题，修改后自检
6.6.3 内部审核：
  - 同事观看并反馈
  - 记录问题
6.6.4 根据反馈修改
6.6.5 修改后再次自检
6.6.6 用户审核：
  - 发送给用户审核
  - 等待反馈
6.6.7 如有反馈，记录并修改
6.6.8 修改完善
6.6.9 最终确认
6.6.10 准备发布
```

### Node 7: 平台发布（17动作）

```
7.7.1 准备发布内容：
  - 标题/描述/标签
7.7.2 适配平台要求：
  - 尺寸/时长/格式
7.7.3 编写文案：
  - 引导评论/点赞
7.7.4 选择发布时间：
  - 黄金时段推荐
7.7.5 登录抖音：
  - 上传视频
  - 填写信息
  - 设置封面
  - 添加话题
  - 发布
7.7.6 登录视频号：
  - 重复流程
7.7.7 登录B站：
  - 重复流程
7.7.8 登录小红书：
  - 重复流程
7.7.9 记录各平台发布状态
7.7.10 检查发布成功
7.7.11 如有问题，及时处理
7.7.12 关注审核状态
7.7.13 如被退回，修改后重新发布
7.7.14 记录发布时间和链接
7.7.15 分享到社群
7.7.16 跟进评论回复
7.7.17 更新发布进度
```

### Node 8: 数据复盘（12动作）

```
8.8.1 统计抖音数据：
  - 播放量/点赞/评论/转发/收藏
8.8.2 统计视频号数据：
  - 播放量/点赞/评论/转发
8.8.3 统计B站数据：
  - 播放量/点赞/投币/收藏/评论
8.8.4 统计小红书数据：
  - 播放量/点赞/收藏/评论
8.8.5 对比各平台数据：
  - 找出表现最好平台
  - 分析原因
8.8.6 分析内容表现：
  - 完播率
  - 点赞率
  - 评论率
8.8.7 分析传播路径：
  - 推荐流量/搜索流量/分享流量
8.8.8 分析案源转化：
  - 咨询数/转化数
8.8.9 总结教训：
  - 做得好的是什么
  - 需要改进的是什么
8.8.10 识别优秀元素：
  - 可复用的脚本结构
  - 可复用的表现形式
8.8.11 输出报告：
  - 数据报告
  - 改进建议
8.8.12 指导下次策划
```
